# GoldieOS App Development Guide

## Overview

GoldieOS is an embedded RTOS platform targeting HiSilicon WS63 (RISC-V) with a 320x240 display, touch input, GPIO keyboard, SLE (StarLink Enhanced) wireless, WiFi, SD card, battery, and audio codec. Apps are written in C++ using the tinyui GUI framework and service-based architecture.

---

## 1. Project Structure

```
goldieos/
  include/
    core/           # Core framework headers (services, app manager, drivers)
    gui/            # tinyui GUI widget headers
    osal/           # OS abstraction layer
    services/       # Service-specific headers (alarm, ntp, aualgo)
  apps/             # All application code goes here
  sdk_integration/  # AI SDK bridge (convai_bridge)
  platform/         # Platform abstraction layer
  services/         # Service implementations
```

**Each app directory** contains:
- `main_app.cpp` — App logic, lifecycle, service usage, business code
- `main_ui.h` — Auto-generated UI (from tinyui tool), declares all widgets statically
- `assets/` — Image sprites as C header files (RGB565 format)

## 2. App Lifecycle Contract

Every app MUST define these 6 functions and register itself via `install_app()`:

### 2.1 App_t Structure

```cpp
typedef struct App_t {
    char app_name[32];          // Application name (max 32 chars)
    void (*h_app_run)(void);    // Called when app is launched
    void (*h_app_exit)(void);   // Called when app is closed
    void (*h_app_suspend)(void);// Called when app goes to background
    void (*h_app_resume)(void); // Called when app comes to foreground
    void (*h_touch_event)(int,int,int);  // (pressure, x, y)
    void (*h_keyboard_event)(int,int);   // (pressure, keycode)
    uint16_t* icon;             // 56x56 RGB565 icon for launcher (NULL if no icon)
    int app_status;             // Managed by framework, init to 0
} App_t;
```

### 2.2 Registration Pattern (REQUIRED)

```cpp
// Step 1: Define lifecycle functions as static
static void goldie_app_run(void) { /* ... */ }
static void goldie_app_exit(void) { /* ... */ }
static void goldie_app_suspend(void) { /* ... */ }
static void goldie_app_resume(void) { /* ... */ }
static void goldie_touch_event(int pressure, int x, int y) {
    if (Window_main) Window_main->handleEvent(pressure, x, y);
}
static void goldie_keyboard_event(int pressure, int key);

// Step 2: Define the App_t instance
static App_t my_app = {
    "我的应用",                // app_name
    goldie_app_run,            // h_app_run
    goldie_app_exit,           // h_app_exit (mandatory)
    goldie_app_suspend,       // h_app_suspend (mandatory)
    goldie_app_resume,        // h_app_resume (mandatory)
    goldie_touch_event,       // h_touch_event (mandatory)
    goldie_keyboard_event,   // h_keyboard_event (mandatory)
    (uint16_t*)my_icon_56_56, // icon (can be NULL)
};

// Step 3: Registration entry function + auto-init macro
static void my_app_entry(void) {
    install_app(&my_app);
}
GOLDIE_INIT_CALL_(my_app_entry);
```

### 2.3 Lifecycle Implementation Template

```cpp
static void goldie_app_run(void) {
    // 1. Get required services
    // 2. Initialize UI (main_ui_init())
    // 3. Set up callbacks
    // 4. Create threads if needed
    // 5. Set running flags
}

static void goldie_app_exit(void) {
    // 1. Set running flag to 0 to stop threads
    // 2. Stop audio playback/recording
    // 3. Close file handles
    // 4. Wait for threads to exit (check exit flags)
    // 5. Destroy threads
    // 6. Unregister callbacks
    // 7. window_exit() (cleans up UI widgets)
    // 8. Resume paused services (e.g. AualgoService)
}

static void goldie_app_suspend(void) {
    // 1. Stop animations / stop flags
    // 2. window_suspend() (hides window)
}

static void goldie_app_resume(void) {
    // 1. window_resume() (shows window)
    // 2. Restart animations / refresh UI
}

static void goldie_keyboard_event(int pressure, int key) {
    if ((key == SYSTEM_KEY_VALUE_BACK) && (pressure == 1)) {
        goldie_exit_app(&my_app);
    }
    // WAKEUP key (301) can be used for app-specific shortcuts
    if ((key == SYSTEM_KEY_VALUE_WAKEUP) && (pressure == 1)) {
        // Handle wakeup action
    }
}
```

## 3. main_ui.h — Auto-Generated UI File

`main_ui.h` is generated by the tinyui GUI design tool and should NOT be manually modified for widget structure. It provides:

### 3.1 What main_ui.h Contains

```cpp
// Includes all widget headers, image assets, boundary mask headers
// Declares all widgets as static std::shared_ptr<WidgetType>
// Provides three lifecycle functions:

static void main_ui_init();    // Creates all widgets, sets hierarchy, flushes
static void window_exit();     // Cleans up ALL widgets (reset shared_ptrs)
static void window_suspend();  // Window_main->setVisible(false)
static void window_resume();   // Window_main->setVisible(true) + flush
```

### 3.2 Handling Missing main_ui.h

When creating a new app without the tinyui tool, manually write `main_ui.h` following this pattern:

```cpp
#ifndef INCLUDE_MAIN_UI_H
#define INCLUDE_MAIN_UI_H
#include <memory>
#include "goldie_display.h"
#include "window.h"
#include "frame_view.h"
#include "label_view.h"
#include "button_view.h"
// ... other widget headers and asset headers ...

// Declare global widget variables
static std::shared_ptr<LabelView> LabelView_title;
static std::shared_ptr<ButtonView> Button_ok;
static std::shared_ptr<FrameView> FrameView_main;
static std::shared_ptr<Window> Window_main;

static void main_ui_init() {
    // Create widgets with std::make_shared<WidgetType>(width, height)
    // Window: std::make_shared<Window>(APP_WINDOW_START_X, APP_WINDOW_START_Y, width, height)
    Widget = std::make_shared<WidgetType>(w, h);
    Widget->setColor(0xFFFF);       // Background color (RGB565)
    Widget->setTextColor(0x0000);   // Text color
    Widget->setVisible(true);
    // Optional: setImageBuffer, setBoundaryBuffer, setMaskBuffer

    // Build hierarchy: parent->addView(child, x, y)
    FrameView_main->addView(LabelView_title, 10, 20);
    Window_main->addView(FrameView_main, 0, 0);

    // Flush to display
    Window_main->flush(0, 0, APP_WINDOW_WIDTH, APP_WINDOW_HEIGHT);
}

static void window_exit() {
    Window_main->setVisible(false);
    Window_main->removeAllViews();
    FrameView_main->removeAllChilds();
    // Reset ALL shared_ptrs
    Widget.reset();
    Window_main.reset();
}

static void window_suspend() {
    Window_main->setVisible(false);
}

static void window_resume() {
    Window_main->setVisible(true);
    Window_main->flush(0, 0, APP_WINDOW_WIDTH, APP_WINDOW_HEIGHT);
}
#endif
```

## 4. System Services API Reference

Services are obtained via `wait_service(SERVICE_INDEX)` (blocking) or `get_service(SERVICE_INDEX)` (non-blocking, may return NULL).

### 4.1 Service Index Enum

```cpp
enum SERVICE_INDEX {
    AUDIO_SERVICE_INDEX     = 0,
    EVENT_SERVICE_INDEX     = 1,
    WIFI_SERVICE_INDEX      = 2,
    SDCARD_SERVICE_INDEX    = 3,
    CLOUD_SERVICE_INDEX     = 4,  // Deprecated, use convai_bridge
    AUALGO_SERVICE_INDEX    = 5,
    SLE_SDP_SERVICE_INDEX   = 6,
    NTP_SERVICE_INDEX       = 7,
    ALARM_SERVICE_INDEX     = 8,
    SLE_WTP_SERVICE_INDEX   = 9,
    MAX_SERVICE_INDEX,
};

void* wait_service(int service_index);  // Blocks until service is ready
void* get_service(int service_index);   // Returns NULL if not ready
```

### 4.2 AudioService

```cpp
AudioService *audio = (AudioService*)wait_service(AUDIO_SERVICE_INDEX);

// Audio I/O (stereo 16-bit PCM)
audio->audio_input_config(int flags);           // Configure input
audio->audio_read(void* buffer, unsigned int len);  // Read PCM (returns bytes read)
audio->audio_output_config(int flags);          // Configure output
audio->audio_write(const void* buffer, unsigned int len); // Write PCM

// Playback control
audio->play_start();    // Start audio output stream
audio->play_stop();     // Stop audio output stream
audio->record_start();  // Start audio input stream
audio->record_stop();   // Stop audio input stream

// Ringtones (ring_id: NOTIFY=0, POWERON=1, WAKEUP=2, SLEEP=3, ALARM=4)
audio->play_ring(int ring_id, int loop);  // Play ringtone (loop: 0=once, 1=loop)
audio->stop_ring();
audio->wait_ring();      // Block until ring finishes

// Volume (stream_type: SYSTEM=0, USER=1)
audio->set_volume(int stream_type, float volume);  // volume: 0.0 ~ 1.0
float vol = audio->get_volume(int stream_type);

// VAD (Voice Activity Detection) and algorithm stream
int is_speech = audio->vad_detect(short* buffer, unsigned int len);
audio->algostream_start();
audio->algostream_stop();
audio->reset_playbuffer();
audio->algostream_read(void* buffer, unsigned int len);
unsigned int valid = audio->get_valid_length(int* flag);
```

**Audio Pipeline:**
- Recording: hardware → stereo 16-bit PCM → `audio_read()` (typically 8kHz, 80ms frames)
- Playback: `audio_write(PCM data)` → hardware
- Common consts: `SAMPLERATE=8000, FRAME_MS=80, BYTES_PER_FRAME=2`
- Buffer size: `2 * BYTES_PER_FRAME * FRAME_MS * SAMPLERATE / 1000 = 2560 bytes`

### 4.3 EventService

```cpp
EventService *evt = (EventService*)wait_service(EVENT_SERVICE_INDEX);

// Broadcast (inter-app/inter-service messaging)
int8_t ret = evt->register_broadcast_recv(uint16_t broadcast_id, boardcast_callback cb);
evt->send_broadcast(BroadcastMessage* msg);

// Broadcast IDs:
//   EVENT_BROADCAST_ADDMSG (1)    - New message added
//   EVENT_BROADCAST_INPUT_METHOD (2)
//   EVENT_BROADCAST_POWER (3)     - Battery power update  
//   EVENT_BROADCAST_TIMERUPDATE (4) - Time update (msg contains struct tm*)
//   EVENT_BROADCAST_SHUTDOWN (5)  - System shutting down

// BroadcastMessage structure:
typedef struct {
    uint16_t id;
    void* msg;       // Pointer to data
    uint16_t msg_len;
    unsigned int ext[4];  // Extension fields
} BroadcastMessage;

// Key events:
evt->send_keyevent(int keycode);
evt->register_keyevt_cb(int keycode, KeyEventHandler handler);

// Key codes:
//   EVENT_KEY_CODE_WAKEUP (1), CLOUD_SLEEP (2), CLOUD_RESTART (3),
//   CLOUD_STOP (4), START_RECORD (5), WAKEUP_AIATLK (6), TTS (7),
//   NEW_MSG (8), START_PLAY (9), FINISH_PLAY (10), PLAY (11),
//   VOLUME_DOWN (12), VOLUME_UP (13), PLAY_NEXT (14), STOP (15),
//   SDCARD_INSERT (16), SDCARD_REMOVE (17), HEADSET_INSERT (18),
//   HEADSET_REMOVE (19), SLE_WTP_EVENT (20)
```

```cpp
// SLE Status types: DISABLED, ENABLED, SCANNING, SCAN_DONE, ANNOUNCING,
//                   ANNOUNCE_STOPPED, DEVICE_LIST_UPDATED,
//                   CONNECTION_LIST_UPDATED, PAIRED_STATUS_UPDATED

// SLE Event types: ENABLE, DISABLE, START_SCAN, STOP_SCAN, CONNECT,
//                  DISCONNECT, SEND_DATA, RECEIVE_DATA, START_ANNOUNCE,
//                  STOP_ANNOUNCE, PAIR, UNPAIR, SLAVE_PAIRED
#endif
```

## 6. OSAL (OS Abstraction Layer)

```cpp
#include "goldie_osal.h"

// Threads
void* thread = goldie_thread_create(
    int (*handler)(void* data),  // Thread function
    void* data,                  // Argument to pass
    "thread_name",               // Name for debugging
    unsigned int stack_size      // Stack size (typically 0x1000 = 4KB)
);
goldie_thread_set_priority(void* thread, unsigned int priority); // 0-31, lower=higher
goldie_thread_destroy(void* thread);
goldie_thread_lock();    // Take scheduler lock
goldie_thread_unlock();  // Release scheduler lock

// C++ Thread wrapper (preferred in C++ apps)
#include "goldie_thread.h"
// Constructor: Goldie_Thread(handler_func, data, stack_size)
Goldie_Thread* t = new Goldie_Thread(play_task, NULL, 0x1000);
delete t;  // Destructor stops and cleans up

// Mutex
goldie_mutex mutex;
goldie_mutex_init(&mutex);
goldie_mutex_lock(&mutex);
goldie_mutex_unlock(&mutex);
goldie_mutex_destroy(&mutex);

// Semaphore
goldie_sem sem;
goldie_sem_init(&sem);
goldie_sem_wait(&sem);
goldie_sem_post(&sem);
goldie_sem_destroy(&sem);

// Sleep
goldie_msleep(int milliseconds);

// Memory
void* ptr = goldie_malloc(unsigned long size);
goldie_free(void* addr);

// Time
goldie_timeval tv;
```

## 9. File I/O (FatFs)

GoldieOS uses FatFs for SD card file operations. Include `ff.h`.

```cpp
#include "ff.h"

// Paths always start with "0:/" for SD card
FIL file;
UINT bytes;

// Open/Create
FRESULT res = f_open(&file, "0:/myfile.wav", FA_READ);
res = f_open(&file, "0:/myfile.wav", FA_WRITE | FA_CREATE_ALWAYS);
res = f_open(&file, "0:/myfile.wav", FA_WRITE | FA_OPEN_APPEND);

// Read/Write
f_read(&file, buffer, size, &bytes_read);
f_write(&file, buffer, size, &bytes_written);
f_lseek(&file, offset);   // Seek to position
f_size(&file);            // Get file size

// Close
f_close(&file);
memset(&file, 0, sizeof(FIL));  // Clear handle after close

// Directory operations
DIR dir;
FILINFO info;
f_opendir(&dir, "0:/music/");
while (f_readdir(&dir, &info) == FR_OK && info.fname[0]) {
    // info.fname contains filename
    // info.fattrib & AM_DIR → it's a directory
}
f_closedir(&dir);
```

### 11.2 Multi-Page App Pattern

Use multiple `FrameView` instances, toggle visibility for page switching:

```cpp
static void show_page(int page) {
    FrameView_page1->setVisible(page == 0);
    FrameView_page2->setVisible(page == 1);
    FrameView_page3->setVisible(page == 2);

// In keyboard event, check current page for back navigation:

        if (current_page == 0) {
            goldie_exit_app(&my_app);  // Exit on main page
        } else {
            show_page(--current_page); // Go back
        }
    }
}
```

### 11.3 Background Thread Pattern

```cpp
static volatile int running_flag = 0;
static volatile int thread_exit_flag = 0;
static volatile int thread_busy_flag = 0;

static int worker_thread(void* arg) {
    while (running_flag) {
        if (work_needed) {
            thread_busy_flag = 1;
            // Do work...
            thread_busy_flag = 0;
        }
        goldie_msleep(50);
    }
    thread_exit_flag = 1;
    return 0;
}

// In goldie_app_run():
running_flag = 1;
void* handle = goldie_thread_create(worker_thread, NULL, "worker", 0x1000);

// In goldie_app_exit():
running_flag = 0;
while (!thread_exit_flag) goldie_msleep(100);
goldie_thread_destroy(handle);
```

### 11.4 Audio App Pattern

```cpp

    while (running_flag) {
        if (is_recording) {
            record_busy = 1;
            int len = audio->audio_read(buf, BUFFER_SIZE);
            if (len > 0) {
                // Process audio data...
            }
        } else {
            record_busy = 0;
            goldie_msleep(30);
        }
    }
    goldie_free(buf);
    thread_exit_flag = 1;
    return 0;
}

// In goldie_app_exit():
running_flag = 0;
while (record_busy) goldie_msleep(100);
while (!thread_exit_flag) goldie_msleep(100);
audio->record_stop();
audio->play_stop();
goldie_thread_destroy(thread_handle);
// Resume algorithm service
if (algo) algo->run();
```

## 12. Key Coding Rules and Conventions

### 12.1 Always Do
1. **Include `extern "C" {}`** when including C headers from C++ files
2. **Clean up threads** in `goldie_app_exit()`: set running flag→wait for exit flag→destroy
3. **Pause AualgoService** before audio recording/playback, **resume** on exit
4. **Check `f_close()` then `memset(&file, 0, sizeof(FIL))`** after every file close
5. **Lock mutexes** when accessing shared state across threads
6. **Always implement all 6 lifecycle functions** (run, exit, suspend, resume, touch, keyboard)
7. **Handle BACK key (300)** — typically exit the app on `pressure==1`
8. **Register broadcast receivers** via `EventService` for system events (time updates, power)

### 12.2 Never Do
1. **Never modify** `main_ui.h` widget structure manually (it's auto-generated)
2. **Never call** `goldie_thread_destroy()` before the thread has exited
3. **Never leave** file handles open — always f_close before app exits
4. **Never block** the main thread in `goldie_app_run()` or event handlers
5. **Never skip** mutex unlock on return paths

### 12.3 Memory Management
- Use `goldie_malloc()` / `goldie_free()` for dynamic allocations
- Thread stack sizes: 0x1000 (4KB) is typical, 0x2000 for heavy audio threads
- Buffer sizes for audio: `2 * BYTES_PER_FRAME * FRAME_MS * SAMPLERATE / 1000` (e.g. 2560 for 8kHz/80ms stereo)
- Playback buffers: 2048 bytes typical

### 12.4 Asset Images
- Images are included as C header files containing `const uint16_t` arrays in RGB565 format
- Include with: `#include "rgb16_image_name_w_h.h"`
- Use with: `widget->setImageBuffer((uint16_t*)&rgb16_image_name_w_h)`
- Boundary masks for clipping: `widget->setBoundaryBuffer(boundary_data)`
